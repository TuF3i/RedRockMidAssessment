# 学生选课系统 - 前端应用

## 项目简介

这是一个基于HTML、CSS、JavaScript开发的学生选课系统前端应用，采用蓝白色调设计，支持学生和管理员两种角色登录。

## 功能特性

### 学生端功能
- **用户认证**：登录、注册、退出登录
- **选课管理**：
  - 查看可选课程
  - 选择课程
  - 查看已选课程
  - 退选课程
- **个人信息**：
  - 查看个人信息
  - 修改个人信息

### 管理员端功能
- **学生管理**：
  - 查看学生列表
  - 创建学生账号
  - 编辑学生信息
  - 删除学生账号
  - 查看学生选课情况
- **课程管理**：
  - 查看所有课程
  - 创建新课程
  - 编辑课程信息
  - 删除课程
- **选课控制**：
  - 开始选课
  - 停止选课

## 技术特性

- **响应式设计**：支持桌面端和移动端
- **数据校验**：前端实时数据校验
- **错误处理**：友好的错误提示
- **用户交互**：模态框、确认对话框
- **数据持久化**：使用localStorage保存登录状态

## 部署说明

### 1. Hertz框架部署

将前端文件放在Hertz框架的静态文件目录中：

```go
// 在Hertz路由中添加静态文件服务
h.Static("/", "./static")
```

确保将HTML、CSS、JS文件放在Hertz的静态文件目录下。

### 2. 其他Web服务器部署

也可以使用任何支持静态文件的Web服务器：

#### Nginx配置示例：
```nginx
server {
    listen 80;
    server_name localhost;
    
    location / {
        root /path/to/your/frontend/files;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

#### Python简单HTTP服务器：
```bash
cd /path/to/your/frontend/files
python -m http.server 8000
```

### 3. 开发环境运行

直接在浏览器中打开 `index.html` 文件即可运行。

## 使用说明

### 首次使用

1. 访问应用首页
2. 点击"立即注册"创建学生账号
3. 使用注册的账号登录
4. 选择学生或管理员角色

### 数据校验规则

- **学号**：必须是10位数字
- **姓名**：
  - 长度2-15个字符
  - 只能包含字母、数字、中文和下划线
  - 不能以下划线开头或结尾
  - 不能使用系统保留字（admin, root, user, api, bob）
- **班级**：长度3-15个字符
- **密码**：至少6位
- **年龄**：16-30岁

### 错误处理

当API调用失败时，系统会显示包含状态码和错误信息的模态框，帮助用户了解问题所在。

## 开发说明

### 添加新功能

1. 在对应的HTML文件中添加UI元素
2. 在 `styles.css` 中添加样式
3. 在 `script.js` 中添加交互逻辑和API调用

### 修改主题颜色

所有颜色配置都在 `styles.css` 文件中，主要颜色变量：

- 主色调：`#1976d2` (蓝色)
- 背景色：`#e0f2ff` (浅蓝)
- 错误色：`#f44336` (红色)
- 成功色：`#4caf50` (绿色)

## 注意事项

1. 确保后端API服务已启动并运行在配置的正确地址
2. 浏览器需要支持ES6语法和现代CSS特性
3. 建议使用HTTPS协议部署生产环境
4. 定期清理localStorage中的过期token
